# 服务器部署文件说明

此文档记录服务器端配置的参考结构。当前仓库中的 `server_config/protos/fsae_telemetry.proto` 是本地、服务器、STM32 和 local_sim 共同使用的唯一协议源；线上实际部署仍以 `bitfsae-com:~/fsae_project` 为准。

注意查看 "\Protobuf\server_config"下文件，目录结构参考：
```
server_config/
├── docker-compose.yml
├── protos/
│   ├── fsae_telemetry.options
│   └── fsae_telemetry.proto
└── telegraf/
    └── telegraf.conf
```

## 1. docker-compose.yml

```yaml
services:
  # 1. MQTT Broker
  mosquitto:
    image: eclipse-mosquitto
    container_name: fsae_mosquitto
    ports:
      - "1883:1883"
      - "9001:9001"
    volumes:
      - ./mosquitto/config/mosquitto.conf:/mosquitto/config/mosquitto.conf
      - ./mosquitto/data:/mosquitto/data
      - ./mosquitto/log:/mosquitto/log
    restart: always

  # 2. InfluxDB 1.8
  influxdb:
    image: influxdb:1.8
    container_name: fsae_influxdb
    ports:
      - "8086:8086"
    volumes:
      - ./influxdb_1.8_data:/var/lib/influxdb
    environment:
      - INFLUXDB_DB=fsae_db
    restart: always

  # 3. Telegraf
  telegraf:
    image: telegraf
    container_name: fsae_telegraf
    volumes:
      # 配置文件挂载
      - ./telegraf/telegraf.conf:/etc/telegraf/telegraf.conf:ro
      # Protos 定义文件挂载 (只保留这一行正确的相对路径即可)
      - ./protos:/etc/telegraf/protos:ro
    depends_on:
      - influxdb
      - mosquitto
    restart: always

  # 4. Grafana
  grafana:
    image: grafana/grafana
    container_name: fsae_grafana
    environment:
      - GF_DASHBOARDS_MIN_REFRESH_INTERVAL=200ms
      - GF_SERVER_DOMAIN=bitfsae.xin
      - GF_SERVER_ROOT_URL=https://bitfsae.xin/monitor/
      - GF_SERVER_SERVE_FROM_SUB_PATH=true
    volumes:
      - ./grafana_data:/var/lib/grafana
    restart: always

  # 5. Web (Nginx)
  web:
    image: nginx:alpine
    container_name: fsae_web
    ports:
      - "80:80"   # HTTP
      - "443:443" # HTTPS (新增)
    volumes:
      # 挂载网页代码
      - ./www:/usr/share/nginx/html
      # 挂载自定义的 Nginx 配置 (覆盖默认配置)
      - ./nginx/conf.d:/etc/nginx/conf.d
      # 挂载证书文件
      - ./nginx/cert:/etc/nginx/cert
    restart: always
```

## 2. telegraf/telegraf.conf

注意：Telegraf 使用 `xpath_protobuf` 插件来解析 Protobuf 数据。如果修改了 `.proto` 文件中的字段名，**必须**同步修改这里的 XPath 映射，否则数据无法写入数据库。

当前链路使用**单 Topic**方案：

*   客户端统一发布到 `fsae/telemetry`
*   第一段 `mqtt_consumer` 采集 STM32 旧顶层字段、BMS 摘要字段、IVT-S/能量计字段和 GPS/IMU `motion` 字段，写入 `telemetry`
*   新增的 `motor_state`、`thermal_summary`、`alarm_state` measurement 也都从同一个 `TelemetryFrame` 中拆出来
*   第二段 `mqtt_consumer` 仍然订阅 `fsae/telemetry`，但通过 `metric_selection = "//modules"` 拆出 `bms_data`

### 2.1 当前文档和实现的边界

*   `bms_data` 不是独立 Topic，而是 Telegraf 从同一个 `TelemetryFrame.modules` 中拆出来的 measurement
*   当前服务器仍读取 `TelemetryFrame` 旧顶层字段 `1~14`，不要删除、reserved 或改名这些字段
*   `modules` 和 `16~25` 的 BMS 摘要字段仍保持旧结构，方便兼容现有拆表逻辑
*   IVT-S 字段使用 `ivt_telemetry = 31`，其中电压通道为 U1：`ivt_voltage_u1_mv`
*   GPS/IMU 字段使用 `motion = 33`，当前写入 `telemetry` measurement 的 `gps_speed_kmh/accel_x_g/accel_y_g/accel_z_g/yaw_rate_dps/yaw_deg`
*   如果修改 `.proto` 字段名、字段类型、字段 ID 或 `modules` 结构，必须同时更新：
  *   车端 Nanopb 生成代码
  *   本文档引用的 `telegraf.conf`
  *   线上 Telegraf 容器配置
