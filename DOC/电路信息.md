# 电路与固件配置摘要

本文件记录当前 G473 版本固件使用到的硬件连接和 CubeMX 配置要点。旧 F405 板子的资料只作为
协议迁移背景，不作为本工程引脚依据。

## 主控与时钟

- MCU：`STM32G473RCT6`，LQFP64。
- HSE：`8 MHz`。
- SYSCLK：`160 MHz`。
- FDCAN kernel clock：`40 MHz`，来自 PLLQ。
- 调试接口：SWD，`PA13=SWDIO`，`PA14=SWCLK`。
- `PB4` 用作 `FDCAN3_TX`，固件启动时调用 `HAL_PWREx_DisableUCPDDeadBattery()`。

## CAN

| 名称 | 控制器 | 引脚 | 速率 | 说明 |
| --- | --- | --- | --- | --- |
| CANA | FDCAN1 | PA11 / PA12 | 500 kbit/s | 片内 FDCAN，当前只初始化 |
| CANB | FDCAN2 | PB12 / PB13 | 500 kbit/s | 旧 F405 `CAN2` 替代口 |
| CAN1 | FDCAN3 | PB3 / PB4 | 250 kbit/s | 旧 F405 `CAN1` 替代口 |
| CANC | MCP2518FD + SPI1 | PA4 / PA5 / PA6 / PA7 + PC4 | 500 kbit/s | 外部 CAN，驱动待实现 |

当前三路片内 FDCAN 均配置为 Classic CAN、FIFO0 接收、标准帧和扩展帧全通。

## RS485 与串口

- `USART2` 是 DTU RS485 口：
  - `PA1=USART2_DE`
  - `PA2=USART2_TX`
  - `PA3=USART2_RX`
  - `115200 8N1`
- G473 版本使用 `HAL_RS485Ex_Init()` 配置硬件 DE。
- 旧 F405 的 `PA4 RS485_DIR` GPIO 方向控制不适用于本工程。
- `USART1` 是 CH340X 串口：
  - `PA9=USART1_TX`
  - `PA10=USART1_RX`
  - `115200 8N1`
  - 应用层调试 CLI 默认关闭。

## GPIO

- `PC8=LED0`，高电平点亮。
- `PA4=MCP2518_CS`，默认高电平。
- `PC4=MCP2518_INT`，下降沿 EXTI。
- `PB8=BOOT0_IN`，配置为输入。

## 业务映射

- 旧 F405 `CAN1` 输入链路迁移到 `FDCAN3 / CAN1`。
- 旧 F405 `CAN2` 输入链路迁移到 `FDCAN2 / CANB`。
- `FDCAN1 / CANA` 当前保留为可接收通道，但未绑定旧业务解析。
- `MCP2518FD / CANC` 需要独立驱动，不能套用 STM32 HAL FDCAN API。
